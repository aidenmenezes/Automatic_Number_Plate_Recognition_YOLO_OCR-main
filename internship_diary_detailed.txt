================================================================================
INTERNSHIP DIARY: AUTOMATIC NUMBER PLATE RECOGNITION (ANPR) PROJECT
PERIOD: APRIL 13th, 2026 - MAY 23rd, 2026 (36 WORKING DAYS)
================================================================================

--------------------------------------------------------------------------------
WEEK 1: ORIENTATION, RESEARCH, AND ENVIRONMENT SETUP
--------------------------------------------------------------------------------

DAY 01: April 13 (Monday)
- Work Description: Attended project orientation. Defined the scope of the ANPR system. Conducted initial research on the history and evolution of license plate recognition systems globally.
- Detailed Tasks: Studied the current system architecture used in smart parking systems. Defined key performance indicators (KPIs) like detection accuracy and processing speed.
- Learnings: Learned that ANPR is a multi-stage process involving detection, segmentation, and recognition. Identified the primary challenge: variations in plate designs and lighting conditions.

DAY 02: April 14 (Tuesday)
- Work Description: Deep dive into Object Detection algorithms. Compared YOLO (You Only Look Once) versions with Faster R-CNN and SSD.
- Detailed Tasks: Read technical papers on YOLOv5 architecture. Analyzed why YOLOv5's CSPDarknet53 backbone is efficient for real-time applications.
- Learnings: Understood the "One-Stage" detection concept where a single neural network predicts bounding boxes and class probabilities directly from full images.

DAY 03: April 15 (Wednesday)
- Work Description: Researching Scene Text Recognition (STR) and OCR technologies.
- Detailed Tasks: Investigated the limitations of traditional Tesseract OCR compared to deep learning-based OCR like EasyOCR and PaddleOCR.
- Learnings: Learned that deep learning OCRs are more robust to noise and font variations because they use CNNs for feature extraction and RNNs/LSTMs for sequence modeling.

DAY 04: April 16 (Thursday)
- Work Description: Initial system setup and dependency management.
- Detailed Tasks: Created a dedicated Conda virtual environment. Installed Python 3.9, PyTorch, and Torchvision. Resolved version conflicts between OpenCV and headless environments.
- Learnings: Gained experience in managing clean development environments using 'requirements.txt' to ensure reproducibility across different machines.

DAY 05: April 17 (Friday)
- Work Description: Hardware acceleration and OpenCV integration.
- Detailed Tasks: Configured NVIDIA CUDA and cuDNN for GPU-accelerated training. Verified GPU availability using 'torch.cuda.is_available()'. Wrote a script to test the webcam interface.
- Learnings: Understood the critical role of GPU acceleration in deep learning, noting that inference speed increases by 10-20x compared to CPU-only processing.

DAY 06: April 18 (Saturday)
- Work Description: Git Workflow and Repository Management.
- Detailed Tasks: Initialized the Git repository. Set up a .gitignore file to exclude large model weights (.pt files) and virtual environment folders. Created the initial project structure.
- Learnings: Learned the importance of modular code structure (dividing the project into ai/, helper/, utils/, and static/ directories) for better maintainability.

--------------------------------------------------------------------------------
WEEK 2: DATA ACQUISITION, LABELING, AND PREPROCESSING
--------------------------------------------------------------------------------

DAY 07: April 20 (Monday)
- Work Description: Dataset Acquisition and Exploration.
- Detailed Tasks: Downloaded the "Car Plate Detection" dataset from Kaggle. Analyzed the distribution of images (sunny, rainy, night-time).
- Learnings: Discovered that dataset diversity is key to model robustness. A model trained only on sunny images will fail during rainy weather.

DAY 08: April 21 (Tuesday)
- Work Description: Annotation Format Conversion.
- Detailed Tasks: Converted PASCAL VOC XML annotations to YOLO format (.txt). Wrote a Python script to automate the normalization of bounding box coordinates.
- Learnings: Learned the YOLO format: [class_id, x_center, y_center, width, height], where all values are normalized between 0 and 1.

DAY 09: April 22 (Wednesday)
- Work Description: Image Preprocessing Pipeline.
- Detailed Tasks: Implemented image resizing while maintaining aspect ratio (letterboxing). Applied Histogram Equalization to enhance contrast in low-light images.
- Learnings: Contrast enhancement significantly improves the visibility of character edges, which is vital for the subsequent OCR stage.

DAY 10: April 23 (Thursday)
- Work Description: Data Augmentation Techniques.
- Detailed Tasks: Used the 'Albumentations' library to apply random rotation, brightness adjustment, and Gaussian noise to the training set.
- Learnings: Learned that data augmentation synthetically increases dataset size and prevents the model from overfitting to specific training samples.

DAY 11: April 24 (Friday)
- Work Description: Data Quality Audit.
- Detailed Tasks: Manually reviewed 500+ images to check for incorrect labels or missing boxes. Removed corrupted images and fixed misaligned annotations.
- Learnings: Quality of data is more important than quantity. One "noisy" label can confuse the model during backpropagation.

DAY 12: April 25 (Saturday)
- Work Description: Final Dataset Preparation.
- Detailed Tasks: Partitioned the data into 80% Training, 10% Validation, and 10% Testing. Generated 'train.txt' and 'val.txt' files for YOLOv5 training.
- Learnings: Understood the role of the validation set in tuning hyperparameters without "leaking" information from the test set.

--------------------------------------------------------------------------------
WEEK 3: YOLO MODEL TRAINING AND PLATE DETECTION
--------------------------------------------------------------------------------

DAY 13: April 27 (Monday)
- Work Description: YOLOv5 Model Loading and Configuration.
- Detailed Tasks: Customized the 'data.yaml' file. Loaded the 'yolov5s.pt' (small) model to balance speed and accuracy for an edge-device deployment scenario.
- Learnings: Learned how to modify YAML files to define classes and paths for custom object detection.

DAY 14: April 28 (Tuesday)
- Work Description: Inference Script Development.
- Detailed Tasks: Wrote 'detect.py' to run the model on static images. Implemented logic to draw bounding boxes and display confidence scores using OpenCV.
- Learnings: Learned to handle the model's output tensor, converting it back to human-readable coordinates on the original image.

DAY 15: April 29 (Wednesday)
- Work Description: Plate Region of Interest (ROI) Extraction.
- Detailed Tasks: Developed a script to crop the detected plate area from the original image. Ensured the crop had a small padding to avoid cutting off character edges.
- Learnings: Precision in cropping is critical; if the detection box is too tight, the OCR might fail to recognize the first or last characters.

DAY 16: April 30 (Thursday)
- Work Description: Hyperparameter Tuning for Detection.
- Detailed Tasks: Tested various Non-Max Suppression (NMS) and Intersection over Union (IoU) thresholds to eliminate duplicate detections of the same plate.
- Learnings: Learned that a higher IoU threshold results in more distinct detections but might miss overlapping objects if not tuned correctly.

DAY 17: May 01 (Friday)
- Work Description: Model Evaluation and Metrics Analysis.
- Detailed Tasks: Calculated mAP (Mean Average Precision) at different thresholds (0.5 to 0.95). Analyzed the Confusion Matrix to see which objects were being misclassified.
- Learnings: Understood that "False Negatives" (missing a plate) are more costly in security apps than "False Positives" (detecting a sign as a plate).

DAY 18: May 02 (Saturday)
- Work Description: Speed Optimization.
- Detailed Tasks: Converted the PyTorch model to TorchScript and ONNX formats for faster inference. Tested FP16 (Half-precision) inference.
- Learnings: Exporting models to optimized formats like ONNX can reduce latency by up to 30%, making the system more responsive.

--------------------------------------------------------------------------------
WEEK 4: OCR INTEGRATION AND TEXT PROCESSING
--------------------------------------------------------------------------------

DAY 19: May 04 (Monday)
- Work Description: EasyOCR Integration.
- Detailed Tasks: Integrated the EasyOCR library. Configured it to detect only English alphanumeric characters to reduce search space and improve speed.
- Learnings: Learned about the CRNN (Convolutional Recurrent Neural Network) architecture used by EasyOCR for sequence-to-sequence recognition.

DAY 20: May 05 (Tuesday)
- Work Description: PaddleOCR Implementation and Comparison.
- Detailed Tasks: Set up PaddleOCR and ran benchmarks against EasyOCR on the same dataset. Evaluated text accuracy and CPU/GPU usage.
- Learnings: Discovered that PaddleOCR is generally faster and more accurate for horizontal text, while EasyOCR handles tilted text slightly better.

DAY 21: May 06 (Wednesday)
- Work Description: Advanced Text Filtering Logic.
- Detailed Tasks: Implemented Regular Expressions (Regex) to validate plate formats (e.g., ensuring a specific pattern of letters and numbers).
- Learnings: Learned how to use Regex to "clean" OCR noise, such as converting a detected '|' to '1' or 'O' to '0' based on position.

DAY 22: May 07 (Thursday)
- Work Description: Image Enhancement for OCR.
- Detailed Tasks: Experimented with adaptive thresholding and morphological operations (dilation/erosion) on the cropped plate images before sending them to OCR.
- Learnings: Morphological dilation can help "thicken" thin characters, making them easier for the OCR engine to identify.

DAY 23: May 08 (Friday)
- Work Description: Perspective Correction (De-skewing).
- Detailed Tasks: Attempted to use the 'Four-Point Transform' to straighten plates that were captured at an angle.
- Learnings: Learned that rectifying the perspective of a plate image dramatically increases the OCR confidence score.

DAY 24: May 09 (Saturday)
- Work Description: Handling Multi-line and High-resolution Plates.
- Detailed Tasks: Updated the logic to detect plates with two rows of characters. Implemented a sorting algorithm to read characters from top-to-bottom and left-to-right.
- Learnings: Character order is essential; simply getting the right characters is not enough if they are in the wrong sequence.

DAY 30: May 16 (Saturday)
